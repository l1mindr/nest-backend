# Glossary

## Access Token

Short-lived JWT stored in the `access_token` HTTP-only cookie. It contains `sub`, `sessionId`, and `exp`. It expires after 15 minutes.

## Active Session

A `Session` row where:

- `isRevoked` is `false`.
- `expiresAt` is in the future.
- `owner.id` matches the authenticated user.

## AppError

Project error class with code, domain, status code, metadata, and message. It is converted to the runtime error response by `GlobalExceptionFilter`.

## CSRF Token

Random token generated by `CsrfService`, stored in readable `csrf_token` cookie, and expected in the `x-csrf-token` header for unsafe methods.

## Data Response Envelope

The global success response shape:

```json
{
  "data": {}
}
```

Implemented by `DataResponseInterceptor`.

## Device Context

Parsed request device information attached by `DeviceMiddleware`. Includes browser name/version, OS name, device type, and simple trust/risk values.

## Feature Module

A Nest module under `src/features` representing a business capability: auth, security, sessions, token, or users.

## Refresh Token

JWT stored in the `refresh_token` HTTP-only cookie. A bcrypt hash of the current refresh token is stored on the server-side `Session` row.

## Rotation

Refresh process that replaces the current refresh token with a new one and updates the session's refresh hash, timestamps, expiry, and version.

## Session Version

Integer field on `Session` used in a conditional database update during refresh rotation.

## User Role

Enum used by `RolesGuard`. Current values are `ADMIN` and `USER`.

## User Status

Enum stored on `User`. Current values are `ACTIVATE`, `DEACTIVATE`, and `SUSPEND`. The current auth flow does not enforce this value.

## Whitelisting

Validation-pipe behavior that strips allowed DTO properties and rejects unknown request fields because `forbidNonWhitelisted` is enabled.
